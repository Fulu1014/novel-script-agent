# 🎬 小说转剧本智能助手 (Novel-to-Script AI Agent)

> 一个基于多 Agent 协同架构与大语言模型（LLM）的工业级文本改编流水线，能够将超长篇小说（TXT/EPUB/PDF）全自动化、无损地转化为标准好莱坞影视剧本格式。

🔗 **在线体验**: [小说转剧本智能助手 · Streamlit](https://novel-script-agent-ucfssdzgznypksbumw2hdp.streamlit.app/) 

---

## 📖 项目简介

在将长篇小说（如《西游记》、《三国演义》及海量网络小说）改编为影视剧本时，传统大模型往往会面临**“上下文遗忘”**、**“剧情摘要化（吞设定）”**、**“场景编号错乱”**等致命痛点。

本项目针对上述痛点进行了深度的架构级优化。通过构建**「统一文本解析引擎」**与**「带动态记忆流的多 Agent 协同系统」**，实现了 1:1 像素级还原的剧本改写。无论是古典名著的晦涩文言，还是网络小说的非标排版，本系统均能提供真正意义上“0 漏戏、0 崩人设、可挂机跑全本”的商业级交付方案。

---

## ✨ 核心功能与亮点

### 1. 🗂️ 多格式统一智能解析引擎
* **全格式兼容**：支持 `.txt`, `.epub`, `.pdf` 文件的上传与解析。
* **抗噪与容错**：内置强悍的复合正则表达式（Regex）与动态容错机制，完美识别中英文数字（如`第一○回`、`Chapter 10`），自动过滤网文常见的“求月票/打赏”等噪音废话。
* **纯净结构化**：抛弃不稳定的 HTML 结构，强行提取纯文本并重构段落，输出极度纯净的 JSON 结构化章节流。

### 2. 🧠 全局设定动态进化流 (Long-term Memory)
* **设定提取**：自动阅读开篇，提取“世界观”、“核心人设”与“剧情脉络”生成《剧组圣经》。
* **边写边记**：每改编完一章，负责“统筹”的 Agent 会自动阅读新章节，将新登场的人物和法宝平滑融入全局设定，彻底解决长文本大模型的“遗忘症”。

### 3. 🔬 显微级无损改编与预言家雷达
* **微米级切片 (Micro-Chunking)**：拒绝大篇幅投喂，将单章强制切分为 800 字的“显微碎片”，彻底扼杀大模型的“摘要/概括”本能，逼迫其丰满动作与对话细节。
* **下文雷达 (Forward Preview)**：在处理当前碎片时，系统会隐蔽下发后文 200 字作为“剧情预告”，防止大模型强行“杀青”导致剧情割裂。

### 4. 🛡️ 工业级防爆破与自动重排
* **场景自动重排**：剥夺大模型容易出错的计数权，剧本生成后由 Python 引擎在毫秒级内自动对 `【场景 X】` 进行全局自增重排，绝对无缝衔接。
* **指数退避与断点续传**：内置强大的 API 重试机制，无惧网络波动与并发限流。系统实时在本地存档设定与剧本，断网刷新后一键“恢复存档”，从断点继续全自动挂机。

## 🛠️ 技术栈

本项目采用了极简但高效的技术栈，确保云端部署的轻量化与高可用性：

* **编程语言**：Python 3.x
* **前端与交互框架**：`Streamlit` (支持实时进度流式渲染与云端一键部署)
* **大语言模型驱动**：基于 `OpenAI SDK` 规范，全面兼容 SiliconFlow 生态大模型（如 `DeepSeek-V2.5`, `Qwen2.5-72B` 等，支持高并发超长上下文）。
* **文档解析矩阵**：
  * `ebooklib` & `BeautifulSoup4`：用于 EPUB 电子书的 DOM 树净化与纯文本榨取。
  * `pdfplumber`：用于 PDF 文档的精准坐标级文本提取。
  * `chardet`：TXT 纯文本的智能编码嗅探。
* **核心文本调度**：Python 原生 `re` (正则表达式) 应用于标题降维打击与场景动态追踪。
