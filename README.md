# 🎬 小说转剧本智能助手 (Novel-to-Script AI Agent)

> 一个基于多 Agent 协同架构、长短记忆双轨制（RAG）与大语言模型（LLM）的文本改编流水线，能够将超长篇小说（TXT/EPUB/PDF）全自动化、无损地转化为标准影视剧本格式。

🔗 **在线体验**: [小说转剧本智能助手 · Streamlit](https://novel-script-agent-ucfssdzgznypksbumw2hdp.streamlit.app/)

---

## 📖 项目简介

在将长篇小说（如《西游记》、《三国演义》及海量网络小说）改编为影视剧本时，传统大模型往往会面临**“上下文遗忘”**、**“剧情摘要化（吞设定）”**、**“场景编号错乱”**以及**“API 频繁断连”**等致命痛点。

本项目针对上述痛点进行了深度的架构级优化。通过构建**「长短记忆双轨制引擎」**与**「带底层网络防护的协同系统」**，实现了 1:1 像素级还原的剧本改写。无论是古典名著的晦涩文言，还是需要保留诗词赋文的复杂场景，本系统均能提供真正意义上“0 漏戏、0 崩人设、防断连”的交付方案。

---

## ✨ 核心功能与硬核亮点

### 1. 🧠 长短记忆双轨制 (State Tracking & RAG)
* **短期核心记忆（皮层）**：自动阅读开篇，提取“世界观与核心人设”生成《剧组圣经》，并在章节推进中动态浓缩更新，防止主线崩塌。
* **长期向量记忆（海马体）**：引入 **LangChain** 与 **FAISS 向量数据库**。系统会将已生成的历史剧本实时向量化落盘。在改编后续章节时，通过语义相似度自动召回相关历史剧本，彻底解决配角遗忘与口吻突变问题。
* **动态数据隔离**：针对不同小说生成独立的物理级 FAISS 命名空间，从底层杜绝多本书籍同时处理时的“串戏”与上下文污染。

### 2. 🔬 显微级切片与镜头语言
* **微米级切片 (Micro-Chunking)**：强制将长文本切分为 800 字的显微碎片，扼杀大模型的“摘要本能”，逼迫其丰满动作与对话细节。
* **专业编剧对齐**：通过深度优化的 Prompt 工程，强制模型使用 `[画外音 (V.O.)]` 吟诵原著诗词，运用 `[蒙太奇 (Montage)]` 推进时间线，并严格限制其对全局设定的重复赘述。

### 3. 🛡️ 底层网络接管与极强容错架构
* **防截断无缝续写**：底层监听 `finish_reason == 'length'` 状态码，在 Token 耗尽腰斩时自动触发内存级断点重连，拼装出完整章节。
* **指数退避与场景重排**：内置 Exponential Backoff 重试算法对抗并发限流；结合 Python 原生正则引擎进行毫秒级场景编号强制自增重排，消除 AI 计数幻觉。

### 4. 🗂️ 多格式统一智能解析引擎
* 支持 `.txt`, `.epub`, `.pdf` 文件的精准纯文本榨取与降噪重构，内置复合正则表达式精准拦截非标网文章节排版。

## 🛠️ 技术栈

* **编程语言**：Python 3.x
* **大模型驱动**：基于 `OpenAI SDK`，全面兼容 SiliconFlow 高并发生态。
* **AI 记忆架构**：`LangChain`, `FAISS` (Facebook AI Similarity Search), `OpenAIEmbeddings`
* **网络与并发控制**：`httpx`, `re` (正则表达式)
* **前端与云原生**：`Streamlit` (支持实时进度流式渲染与 Checkpoint 断点续传)
* **文档解析矩阵**：`ebooklib`, `BeautifulSoup4`, `pdfplumber`, `chardet`
